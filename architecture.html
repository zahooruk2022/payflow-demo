Payment Lifecycle — Step by Step

1
User submits payment
React PaymentForm calls POST /api/payments with sender/receiver accounts, amount (GBP), and description. Vite proxy forwards to Spring Boot on :8080.
React Vite proxy REST
2
Transaction saved — status: PENDING
PaymentController delegates to PaymentService. A Transaction entity is persisted to PostgreSQL with status PENDING. A TransactionEvent DTO is broadcast to /topic/transactions — the UI updates immediately with no polling.
Spring MVC JPA WebSocket
3
Published to RabbitMQ
PaymentProducer publishes the transaction ID to payflow.exchange. RabbitMQ routes it to payment.queue. A dead-letter queue (payment.dlq) captures any processing failures — nothing is silently lost.
RabbitMQ AMQP DLQ
4
Consumer picks up — status: PROCESSING
PaymentConsumer (@RabbitListener) receives the message, loads the full transaction from PostgreSQL, and updates the status to PROCESSING. Another WebSocket broadcast is sent so the UI shows the in-progress state.
@RabbitListener JPA load WS broadcast
5
Fraud detection runs
FraudDetectionService evaluates three rules additively:
• High Amount — amount > £50,000 → +60 risk
• Rapid Succession — 3+ payments in 60s from same account → +70 risk (Redis INCR + EXPIRE)
• Round Number — exact whole-pound amount > £5,000 → +20 risk
Fraud rules Redis INCR Score ≥ 60 → FLAGGED
✗
Score ≥ 60 — FLAGGED
Transaction quarantined. FraudAlert saved to PostgreSQL. Broadcast to /topic/fraud-alerts and /topic/transactions. Fraud Alerts tab badge increments.
FraudAlert persisted UI badge++
✓
Score < 60 — COMPLETED
Account balances updated atomically. Transaction marked COMPLETED. Broadcast to /topic/transactions. KPI cards, chart, and balances panel all update live.
Balances updated Chart tick
6
Stats broadcast — KPIs refresh everywhere
After every completed or flagged payment, PaymentService computes a fresh DashboardStats snapshot (total, completed, flagged, volume, rates) and broadcasts it to /topic/stats. Every connected browser receives the update simultaneously.
DashboardStats All browsers sync

Backend Services

⚙️
PaymentService
service/PaymentService.java
Orchestrates the full payment lifecycle: persist PENDING → publish to RabbitMQ → WebSocket broadcast. Also triggered post-processing to broadcast stats.
Core service
🛡️
FraudDetectionService
service/FraudDetectionService.java
Rule engine: High Amount (+60), Rapid Succession (+70 via Redis), Round Number (+20). Returns a FraudResult with score and alert type. Score ≥ 60 = FLAGGED.
Fraud
📏
PaymentMetricsService
service/PaymentMetricsService.java
Micrometer instrumentation: counters for submitted/completed/flagged/failed payments, volume (GBP), and a Timer with p50/p95/p99 percentiles for processing latency.
Metrics
📥
PaymentConsumer
messaging/PaymentConsumer.java
@RabbitListener on payment.queue. Loads transaction, updates to PROCESSING, invokes FraudDetectionService, saves result (COMPLETED or FLAGGED), broadcasts updates.
RabbitMQ

WebSocket Topics

STOMP over SockJS · connect to http://localhost:8080/ws
/topic/transactions
Fired on every transaction state change: PENDING → PROCESSING → COMPLETED/FLAGGED. Every connected browser receives each event.
"id": "txn-uuid", "reference": "PF-20260617-...", "senderName": "Vantage Bank PLC", "receiverName": "Meridian Bank PLC", "amount": 60000.00, "currency": "GBP", "status": "FLAGGED", "riskScore": 60, "timestamp": "2026-06-17T14:32:01"
/topic/fraud-alerts
Fired when a transaction is FLAGGED. Same TransactionEvent shape — the alert tab badge counter increments and a new card appears in the Fraud Alerts tab.
"id": "txn-uuid", "status": "FLAGGED", "riskScore": 60, "alertType": "HIGH_AMOUNT", "amount": 60000.00, "senderName": "Vantage Bank PLC"
/topic/stats
Broadcast after every completed or flagged payment. All KPI stat cards on every connected browser refresh simultaneously — no page reload required.
"totalTransactions": 42, "completedTransactions": 35, "pendingTransactions": 2, "flaggedTransactions": 5, "totalVolume": 1250000.00, "fraudRate": 11.9, "successRate": 83.33
